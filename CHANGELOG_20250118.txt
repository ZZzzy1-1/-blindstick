## 二、后端 (server/proxy_server.py)

### 当前版本特性

#### 1. TTS传输方案（URL方式）
```
旧方案: 后端合成 → MQTT直接发送音频数据 → ESP32
新方案: 后端合成 → 保存文件 → 生成URL → MQTT发送URL → ESP32下载 → 播放
```

**优点:**
- 避免MQTT传输大文件（>100KB）的问题
- ESP32可以控制下载时机
- 更稳定可靠

**文件命名格式:**
```python
filename = f"tts_{timestamp}_{hash}.wav"  # 例: tts_1234567890_113f28c9.wav
```

#### 2. 主要API接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/tts/push` | POST | 推送TTS到ESP32，返回URL |
| `/api/asr` | POST | 语音识别 |
| `/audio/<filename>` | GET | 下载TTS音频文件 |
| `/health` | GET | 健康检查 |

#### 3. MQTT主题

| 主题 | 方向 | 说明 |
|------|------|------|
| `blindstick/tts/url` | 后端→ESP32 | 发送TTS音频URL |
| `blindstick/sensors` | ESP32→后端 | 传感器数据上报 |

#### 4. 关键配置
```python
MQTT_BROKER = "u72a7838.ala.asia-southeast1.emqxsl.com"
MQTT_PORT = 8883 (TLS)
AUDIO_CACHE_DIR = "./audio_cache"
BAIDU_TTS_PER = "4146"  # 发音人
```

#### 2. I2S引脚配置
```cpp

#define I2S_WS_PIN      5   // LRCK
#define I2S_SCK_PIN     2   // BCLK
#define I2S_SD_PIN      8   // MIC_IN

// 扬声器 (MAX98357)
#define I2S_BCK_PIN     47  // SPK_BCLK
#define I2S_WS_OUT_PIN  41  // SPK_LRCK
#define I2S_DATA_PIN    21  // SPK_OUT
```

#### 3. MQTT订阅主题
```cpp
blindstick/tts/url          // TTS音频URL（新方案）
blindstick/tts/control      // 流式TTS控制
blindstick/tts/stream/+     // 流式TTS音频
blindstick/nav/steps        // 导航步骤
blindstick/tts/request      // TTS请求
```

#### 4. 语音识别流程
```
录音 → WebSocket流式识别 → 收到结果 → 处理命令 → 等待5秒 → 开始下一次识别
```

**关键改进:**
- 处理完结果后立即清空 `asrResult`，避免重复处理
- 导航触发后等待5秒让TTS播报完成
- 支持直接说地点名（无需"带我去"等触发词）

#### 5. TTS播放流程（URL方式）
```cpp

    // 4. HTTP下载音频
    // 5. 播放
    // 6. 恢复语音识别

```

#### 6. 导航功能
- 支持百度地图步行路线规划
- 自动播报导航步骤
- 进度更新（每3秒+5%，约60秒完成一步）

## 六、关键问题修复记录

### 问题1: MQTT TLS连接失败（-2错误）
**原因:** 证书验证失败、时间不同步
**修复:**
- 添加NTP时间同步
- 使用 `setInsecure()` 跳过证书验证
- 增加握手超时时间

### 问题2: TTS音频传输失败
**原因:** MQTT消息太大（>100KB），导致传输失败
**修复:** 改为URL方案，ESP32主动下载

### 问题3: 语音识别重复处理
**原因:** `asrResult` 未清空
**修复:** 处理完后立即清空结果变量

### 问题4: TTS重复播放
**原因:** 前后端都发送了相同消息
**修复:**
- 后端：文件名添加时间戳
- ESP32：基于文本+3秒窗口去重

### 问题5: 导航进度太快
**原因:** 每1秒+10%，10秒就完成一步
**修复:** 改为每3秒+5%，约60秒完成一步

### 问题6: 需要触发词才能导航
**原因:** 只识别"带我去"等触发词
**修复:** 无触发词时直接使用识别文本搜索

## 八、使用说明

### 正常启动流程
1. ESP32开机 → 连接WiFi → 连接MQTT
2. 播放"导盲杖系统启动成功，欢迎使用"
3. 开始语音识别（自动循环）

### 导航使用
1. **大声说目的地**（如"天安门"、"湖北师范大学"）
2. ESP32识别 → 搜索地点 → 规划路线
3. 播报"开始导航到XXX，全程XXX米..."
4. 导航过程中自动播报步骤

### 注意事项
- 说话声音要大，距离麦克风10cm以内
- 导航过程中会暂停语音识别
- TTS播报完成后自动恢复语音识别
